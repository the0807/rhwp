# Task M100-993 Stage 9 계획서

## 1. 목적

Stage 8에서 `tb-org-02` 조직도 셀 줄나눔 문제는 대표 셀 몇 개의 보강이 아니라,
조직도 TABLE 전체 셀의 `LIST_HEADER` contract 보강에서 개선되는 것으로 확인되었다.

개선이 확인된 probe:

```text
07_all_org_cells_list_width_ref_plus_extra.hwp
08_all_org_cells_list_bundle_plus_para_header_tail.hwp
```

Stage 9의 목적은 이 결과를 production adapter 후보로 옮겨, HWPX 출처 `Cell`이 HWP5로
저장될 때 필요한 셀 `LIST_HEADER` contract를 materialize하는 것이다.

## 2. 현재 결론

`tb-org-02`의 문제는 문자열을 다시 자르거나 셀 높이를 줄이는 문제가 아니다.

```text
HWPX 원본:
  이미 2글자 단위 문단 구조를 가진다.

rhwp IR:
  2글자 단위 문단 구조를 유지한다.

generated HWP:
  문단 구조는 유지하지만, 한컴 에디터가 셀 내부 폭을 다르게 해석하여
  한글자씩 줄나눔한다.
```

따라서 수정 방향은 다음이다.

```text
셀 높이 보정 X
문자열 임의 분할 X
HWP5 셀 LIST_HEADER 저장 contract materialization O
```

## 3. 구현 후보

대상 경로:

```text
src/document_core/converters/hwpx_to_hwp.rs
src/model/table.rs
src/serializer/control.rs
```

현재 구조:

```text
Cell {
  list_header_width_ref: u16,
  raw_list_extra: Vec<u8>,
  apply_inner_margin: bool,
  ...
}
```

serializer는 이미 다음을 출력한다.

```text
LIST_HEADER:
  n_paragraphs: u16
  list_attr: u32
  list_header_width_ref: u16
  cell fixed payload: 26 bytes
  raw_list_extra: optional
```

따라서 Stage 9에서는 serializer의 record 구조를 바꾸지 않고, HWPX 출처 adapter에서 `Cell` 값을
보강한다.

## 4. 1차 materialization 규칙

Stage 8의 `07`과 같은 축을 production adapter 후보로 구현한다.

```text
대상:
  HWPX/HWP3 출처 문서를 HWP5로 저장할 때의 모든 표 셀

조건:
  cell.raw_list_extra가 비어 있음

보강:
  1. cell.list_header_width_ref bit 0을 세운다.
     - Stage 8 관찰값: oracle bytes 6..7 = 0x0001
     - 기존 generated bytes 6..7 = 0x0000

  2. cell.raw_list_extra를 13바이트로 생성한다.
     - bytes 0..4 = cell.width (u32 little endian)
     - bytes 4..13 = 0
```

주의:

```text
기존 HWP 출처 문서의 raw_list_extra는 보존한다.
raw_list_extra가 이미 있는 셀은 덮어쓰지 않는다.
```

## 5. 보류 후보

Stage 8의 `08`은 `07`에 셀 내부 `PARA_HEADER` tail을 추가한 후보이다.
작업지시자 판정에서 `08`이 `07`보다 명확히 더 좋다는 추가 증거가 아직 없으므로,
Stage 9에서는 `PARA_HEADER` tail materialization을 production 후보에 포함하지 않는다.

```text
이번 단계 포함:
  LIST_HEADER width_ref/property word + raw_list_extra

이번 단계 제외:
  셀 내부 PARA_HEADER tail 합성
```

`07` 반영 후에도 한컴 줄나눔/표 높이가 충분히 정상화되지 않으면, 다음 단계에서 `08` 축을 별도
계획으로 다룬다.
