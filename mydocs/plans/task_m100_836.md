# Task #836 수행계획서 — Endnote(미주) 본문 렌더링 구현

## 이슈

[Issue #836](https://github.com/edwardkim/rhwp/issues/836) — HWP5 시험지 sample의 페이지 column 높이 측정 부정합. 근본 원인: Endnote 본문 렌더링 부재.

## 배경

### 현재 상태

| 영역 | Footnote(각주) | Endnote(미주) |
|---|---|---|
| 인라인 마커 (위첨자) | ✅ 구현 | ✅ 구현 |
| pagination 영역 확보 | ✅ `page.footnotes.push()` | ❌ 무시 (`_ => {}`) |
| 본문 렌더링 | ✅ `build_footnote_area()` | ❌ 미구현 |

### 한컴의 미주 동작

한컴에서 미주는 각주와 다른 배치 정책:
- **각주**: 해당 페이지 하단에 즉시 배치
- **미주**: `FootnotePlacement` 설정에 따라
  - `EachColumn` (기본) → 문서 마지막에 모아서 출력
  - `BelowText` → 구역(섹션) 마지막에 모아서 출력

`SectionDef.endnote_shape.placement`로 결정.

### 영향 범위

시험지/문제집 HWP 파일에서 Endnote가 풀이/답 텍스트를 담는 용도로 사용. 각 sample에 40~50개 Endnote → 미렌더링 시 페이지 수/레이아웃 부정합.

## 목표

1. Endnote 본문(`en.paragraphs`)을 문서 끝 또는 섹션 끝에 모아서 렌더링
2. Endnote 영역에 구분선 + 번호 + 본문 텍스트 배치
3. 시험지 4종에서 한컴 PDF와 페이지 수 정합

## 구현 방향

### A안: 문서 끝 단일 영역 (최소 구현)

모든 Endnote를 수집하여 마지막 섹션의 마지막 페이지 이후에 새 페이지로 배치. 각주 영역(`build_footnote_area`)과 유사한 패턴이지만 페이지 단위가 아닌 문서 단위.

**장점**: 구현 단순, 대부분의 시험지에서 동작
**단점**: 섹션별 미주 배치(`BelowText`) 미지원

### B안: 섹션 끝 + 문서 끝 분기 (완전 구현)

`endnote_shape.placement`에 따라:
- `BelowText` → 해당 섹션 마지막 페이지 하단 또는 다음 페이지
- `EachColumn` → 전체 문서 마지막

**장점**: 한컴 정합 완전
**단점**: pagination 변경 범위가 큼

### 권고: A안 우선 + B안 후속

## 구현 단계 (최소 3단계)

### Stage 1: Endnote 수집 + pagination 반영

- `typeset.rs`에서 `Control::Endnote` 처리 추가 (현재 `_ => {}`)
- 섹션별 Endnote 목록 수집
- `PageContent`에 endnote 참조 추가 (또는 별도 endnote area)

### Stage 2: Endnote 영역 렌더링

- `layout.rs`에 `build_endnote_area()` 추가 (각주 `build_footnote_area()` 참조)
- 구분선 + 번호 + 본문 paragraphs 렌더
- 여러 페이지에 걸치는 미주 영역 처리

### Stage 3: 회귀 검증 + 시각 판정

- 기존 fixture 회귀 sweep
- 시험지 4종 한컴 PDF 정합 시각 판정
- Endnote 미사용 문서에서 동작 무영향 확인
