## 1. 배경

HWPX 문서에서 수식(`<hp:equation>`)과 글자들이 조합된 경우의 HWP 저장 기능 구현.
Task #1050 (각주) → #1052 (글상자 안 각주) → #1058 (각주 contract 5 라운드) 누적 후
HWPX → HWP 어댑터의 다음 누락 영역.

작업지시자 자료:
- `samples/hwpx/math-001.hwpx` — HWPX 원본 (수식 2개 + 본문)
- `samples/math-001.hwp` — **정답지** (한컴 직접 저장)
- `pdf-large/hwpx/math-001.pdf` — 시각 정답지
- `saved/111math-001.hwp` — **현재 rhwp-studio 저장 결과** (본 task 출발점)

## 2. 사전 조사 결과

| 영역 | 상태 |
|------|------|
| HWPX `<hp:equation>` 파싱 | 정상 (`src/parser/hwpx/section.rs::parse_equation` line 3676-3786) |
| HWP5 'eqed' 파싱 | 정상 (`src/parser/control.rs::parse_equation_control` line 782-828) |
| HWP5 'eqed' 직렬화 | 정상 (`src/serializer/control.rs::serialize_equation_control` line 1884-1916) |
| `Equation` IR struct | 정상 (`src/model/control.rs` line 64-82) — script + font_size + color + baseline + font_name + version_info + raw_ctrl_data |
| HWPX → HWP **어댑터** | **누락** — `src/document_core/converters/hwpx_to_hwp.rs::adapt_paragraph` match 에 `Control::Equation` arm 없음 |

→ 핵심 누락 영역: **어댑터의 수식 처리** + 한컴 호환 contract (Task #1058 의 oracle 방법론 적용).

## 3. 본질 가설 (정답지 vs 저장본 oracle)

Task #1058 의 Stage 4-pivot 방법론을 적용:

```text
HWP → IR → HWP 라운드트립 (samples/math-001.hwp 자체 라운드트립) 이 정상이면
HWPX → IR 만 HWP → IR 과 동일한 IR 을 생성하면 자동으로 한컴 호환.
```

본질 식별 절차:

1. `samples/math-001.hwp` 의 수식 record IR (정답지 oracle) 분석
2. `saved/111math-001.hwp` 의 수식 record IR (현재 저장본) 분석
3. record-level diff → 어댑터 누락 contract 식별
4. 어댑터 정정 + 한컴 시각 판정 게이트

가능한 contract 누락 후보 (#1058 5 라운드에서 학습):
- 수식 control 의 `raw_ctrl_data` (binary payload 보존)
- EQEDIT child record (script + font 정보)
- CommonObjAttr 영역 (`treat_as_char`, `numbering_type=EQUATION`)
- 수식 number 자동 증가 카운터 (numbering)
- 본문 paragraph 의 control_mask bit 설정

## 5. 본 task 의 결정 지점 (작업지시자 확인 필요)
