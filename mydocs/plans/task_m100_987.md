## 배경

`samples/hwp3-sample16.hwp` (HWP 3.0) 쪽 테두리 시각 판정 (작업지시자, 한컴 정답지):

| 항목 | 한컴 정답지 | 현재 rhwp | 원인 위치 |
|------|-------------|-----------|-----------|

| 위치 기준 | **본문(body) 기준** | 종이(paper) 기준 | `src/renderer/layout.rs:944` `paper_based=true` 전역 하드코딩 |

## 회귀 이력 (주의)

`paper_based` 는 [layout.rs:937-943] 에 회귀 history 가 누적된 민감 지점:
- task877: `attr & 0x01 != 0` — sample16 정합, 시험지 회귀
- #920: `attr & 0x01 == 0` — 시험지 정합, sample16 회귀
- #952: `paper_based = true` 전역 — 당시 "모든 sample 정합" 으로 판정

작업지시자 판단: 본 변경은 **HWP 3.0 포맷 한정, 회귀 가능성 낮음**.
단 layout.rs 의 전역 하드코딩을 직접 바꾸면 HWP5/HWPX 시험지에 영향 →
**HWP3 전용 격리** 필수 (CLAUDE.md "HWP3 전용 로직은 `src/parser/hwp3/` 안에서만").

## 설계 방향

### 핵심 원칙

layout.rs 의 `paper_based` 결정 로직을 HWP3 파서가 주입하는 `PageBorderFill.attr`
로 제어하되, HWP5/HWPX 의 기존 `paper_based=true` 동작은 보존한다.

### 변경 지점 (2곳)

1. **`src/parser/hwp3/mod.rs:2798-2803`** — `border_type` → `BorderLineType` 매핑
   - `4 => BorderLineType::Double` 추가 (현재 `_ => Solid`)
   - 기존 1/2/3 (Solid/Dash/Dot) 유지

2. **`src/parser/hwp3/mod.rs:2817-2824` + `src/renderer/layout.rs:944`**
   — HWP3 쪽 테두리를 body 기준으로
   - HWP3 파서가 `PageBorderFill.attr` 에 body-based 의도를 인코딩
   - layout.rs 는 HWPX/HWP5 의 기존 `paper_based=true` 를 깨지 않으면서
     HWP3 가 보낸 신호만 body 로 처리하는 방식 채택
   - 구체 인코딩 방식은 구현계획서에서 확정 (attr 비트 vs 별도 플래그)
