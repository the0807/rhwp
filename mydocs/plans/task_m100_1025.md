# 수행계획서 — #1025: 페이지보다 큰 단일 표 셀 내부 분할 (intra-cell line split)

- 브랜치: `local/task1025` (base: `local/devel` = edf62865)
- 선행: #993 (분할 표 cut 모델), #1022 (측정 정합)

## 1. 문제 정의

페이지보다 큰 **단일 표 셀**이 분할될 때 본문 영역을 넘어 렌더링되거나 용지 밖으로 잘린다.

#993의 RowCut 모델은 **행 단위**로만 cut을 만든다. 한 행(특히 거대 세부내용 셀 + rs=2 라벨 셀)이 페이지 본문보다 크면, 그 행을 통째로 한 페이지에 놓을 수밖에 없어 초과/잘림이 발생한다.

### 확인된 케이스 (요구사항 명세 표 7×3, 비공개 샘플)

| pi | 세부내용 셀(cell[10]) | 표 전체 | 분할 | 본문 초과 |
|----|----------------------|---------|------|----------|
| 321 | ~ | ~ | rows 0..3 / 3..7 | 1.7px (경계) |
| 323 | 904.8px (10문단+내부표) | 927.6px | rows 0..3 / 3..5 / 5..7 | 29px |
| 324 | **1024px** (25문단) | 933.6px | rows 0..3 / 3..5 / 5..7 | **143px, 잘림** |

- cell[6] r=3,c=0 **rs=2** "상세설명" 라벨이 rows 3-4를 걸치고, 거대 세부내용 셀(cell[10], row 4)과 결합하여 첫 조각(rows 0..3)에서 과도하게 밀림.
- 본문 941px(섹션0) / 913px(섹션1)보다 큰 단일 셀은 **행 단위 cut으로는 절대 안 맞음.**

## 2. 목표

단일 셀이 페이지 본문보다 큰 경우, **셀 내부 문단/줄을 페이지 경계로 분할**(intra-cell line-range split)하여 연속 페이지로 흘린다. rowspan 셀이 분할 경계를 가로지르는 경우 포함.

검증 기준: 비공개 184p 문서 `LAYOUT_OVERFLOW` 에서 page-larger 셀 부류(pi=323/324 등) 제거. 기존 분할 표 골든 무회귀.

## 3. 접근 (개요 — 구현계획서에서 단계 확정)

1. **현 RowCut/advance_row_cut 한계 진단**: 행 단위 cut이 page-larger 행을 어떻게 처리하는지, 렌더러가 어디서 잘리는지 정확히 측정.
2. **셀 내부 cut 설계**: 페이지보다 큰 행을 감지 → 해당 행의 셀들을 줄 단위로 페이지 경계까지 채우고 나머지를 다음 페이지로. rs>1 셀의 잔여 높이 추적.
3. **단일 권위 함수 확장**: `advance_row_cut` (또는 신규 보조 함수)에 "행 내부 부분 진행" 개념 추가 — 페이지네이터·렌더러 공유 유지.
4. **렌더러 정합**: `layout_partial_table` 가 셀 내부 cut을 동일하게 그림.
5. **검증·골든**: LAYOUT_OVERFLOW 재측정, 신규 케이스 골든을 한컴 2022 PDF(`pdf/hwpx/...`)로 판정.

## 4. 권위 자료 / 판정

- 시각 판정: 한컴 2022 PDF (macOS/Linux 환경 1차 정답지). 해당 표의 PDF 분할 위치와 대조.
- 한컴이 거대 셀을 어떻게 페이지 분할하는지(셀 내부 줄 분할 여부)가 정답 기준.
