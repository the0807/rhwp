한국어 작성 시 다음 단어 선택 규칙을 따른다:

- "산수" 표현 금지 → "계산" 사용
- 외부 공개 문서(README, CHANGELOG, PR 본문, 스토어 설명)에서 자기검열:
  - 비교/최상급 표현 회피 ("최고의", "유일한", "가장 빠른" 등)
  - 공공기관 오인 표현 회피 (정부/한컴/특정 단체 후원으로 오해될 표현)

**Why**: 한컴 등 기업과 공공기관이 민감한 영역. 정확하고 절제된 어휘가 프로젝트 신뢰도와 외부 노출 안전성을 좌우한다. 작업지시자가 일관되게 grep으로 자기검열을 검증한다.

**How to apply**:
- 문서 작성 시 위 단어들을 자동 회피
- 외부 공개용/내부용 문서를 구분 — 내부용에서 절제 강도 약화 가능
- 모호하면 "계산", "지원", "구현" 같은 중립적 어휘 선택
