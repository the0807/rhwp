# Task M100-1013 계획서

## 1. 목적

`hy-002` 샘플에서 2페이지 글상자 내부 이미지가 한컴 에디터/PDF보다 작게 렌더링되는 문제를
해결한다.

이번 작업은 HWPX의 글상자(`drawText`) 안에 들어 있는 non-TAC 그림의 크기 산정, 내부 여백,
clip/dim, transform 적용 규칙을 한컴 결과와 맞추는 것을 목표로 한다.

## 4. 관찰된 현상

작업지시자 관찰:

```text
rhwp에서 hy-002 2페이지 이미지가 한컴 편집기보다 작게 출력된다.
```

현재 확인된 HWPX 구조상 문제 지점은 글상자 내부 그림이다.

```text
hp:rect
  hp:drawText
    hp:pic treatAsChar="0"
```

따라서 단순 이미지 디코딩 문제가 아니라, 글상자 내부에서 non-TAC 그림을 배치할 때 사용하는
컨테이너 크기, 내부 여백, 그림 표시 크기, clip/dim, rendering matrix 계약을 비교해야 한다.

## 5. Stage 1 작업 범위

Stage 1에서는 구현을 바로 하지 않고, 한컴 기준과 rhwp 기준의 크기 산정 축을 분리한다.

수행 항목:

```text
1. hy-002.hwpx의 2페이지 글상자/그림 XML 구조를 trace한다.
2. 정답 HWP의 대응 shape/picture record bundle을 추출한다.
3. 현재 rhwp IR에서 해당 글상자와 내부 그림의 width/height/clip/dim/margin을 기록한다.
4. 현재 renderer가 실제 배치에 사용하는 컨테이너 크기와 그림 bbox를 기록한다.
5. HWPX의 curSz, sz, orgSz, renderingInfo 중 어떤 값이 한컴 기준 크기와 맞는지 비교한다.
6. hy-001 글상자 내부 이미지 처리와 비교해 #974 회귀 위험을 분리한다.
```

## 6. 우선 검증할 후보 축

이번 단계에서 우선 확인할 후보는 다음이다.

```text
1. hp:rect의 curSz가 hp:sz에 의해 덮어써지는지 여부
2. drawText textMargin을 내부 그림 컨테이너 높이에 잘못 적용하는지 여부
3. non-TAC picture를 글상자 남은 높이로 clamp하면서 과도하게 축소하는지 여부
4. imgClip/imgDim을 출력 bbox 크기로 오해하는지 여부
5. renderingInfo scale matrix를 rect와 pic 양쪽에 중복 또는 누락 적용하는지 여부
```

## 7. 성공 기준

Stage 1 성공 기준:

```text
1. 문제 그림의 HWPX XML 값, IR 값, renderer 배치 값, 정답 HWP record 값을 한 문서에서 비교한다.
2. 이미지가 작아지는 원인이 어느 산정 축에서 발생하는지 1차 후보를 좁힌다.
3. Stage 2에서 적용할 구현 후보를 하나 이상 명확히 제시한다.
```

최종 성공 기준:

```text
1. hy-002 2페이지 글상자 내부 이미지 크기가 한컴 에디터/PDF와 일치한다.
2. hy-001 글상자 내부 TAC/non-TAC 이미지 배치가 회귀하지 않는다.
3. HWPX와 HWP 입력 모두에서 같은 IR 렌더링 계약을 유지한다.
4. 필요한 경우 wasm 빌드 후 작업지시자 시각 판정을 통과한다.
```
