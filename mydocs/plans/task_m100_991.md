## 2. Root cause (KS X 6101 spec 정합 분석)

[mydocs/tech/한글문서파일형식_5.0_revision1.3.md:213-292] HWP5 spec 표 6:
- 확장 컨트롤 (ch=1-3, 11-12, 14-18, 21-23): stream 에 8 wchar 차지, visual 로 1 char position (placeholder)

### HWP3 parser vs HWP5 parser 차이

| | HWP3 | HWP5 |
|---|------|------|
| text 마커 push | `\u{FFFC}` 1 개 / 컨트롤 (line 1567) | **마커 미푸시** (skip) |
| char_offsets entry | 마커 마다 1 entry | 컨트롤 skip → entry 없음 |
| 결과 | text 와 char_offsets 1:1 dense | text 와 char_offsets sparse |

→ HWP5 parser 가 spec 정합 측면에서 marker 누락. composer 가 utf16_range_to_text_range 로 line 별 text 범위 계산할 때 잘못된 매핑.

### pi=394 (HWP5 sample16) 구체적 예

```
3 TAC controls + 3 line_segs:
  [0] 표 "가." (89×7.9 mm) on ls[0] ts=0
  [1] 그림 (161×109 mm) on ls[1] ts=8
  [2] 표 "나." (47×7.9 mm) on ls[2] ts=18

HWP5 IR (현재 버그):
  text="  " (2 chars), char_offsets=[16, 17]
  composer.utf16_range_to_text_range(0, 8): text [0, 0) — 빈!
  composer.utf16_range_to_text_range(8, 18): text [0, 2) — 모든 공백
  → ls[0] 빈 라인으로 처리 → 가. 라벨 위치 어긋남

HWP3 정합 IR (이 fix 의 목표):
  text="\u{FFFC}\u{FFFC}  \u{FFFC}" (5 chars), char_offsets=[0, 8, 16, 17, 18]
  composer 의 line 별 text 범위 정확 매핑
```
